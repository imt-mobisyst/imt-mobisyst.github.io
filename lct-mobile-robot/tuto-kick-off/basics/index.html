Operating System: Linux/Ubuntu for ROS2

ROS stand for Robot Operating System however it extend a classical Operating System, a Linux/Ubuntu OS.

The goal of this tutorial is to come back on basic Operating System notions and mainly the Shell ofering the simplest way to interact with your computer.

  • For an introduction to OS read the wikipedia page

Play with Linux

Ubuntu is a Linux distribution (Operating System + Window Managment + Tools) derived from Debian distribution, in the galaxies of Open source solutions.

  • Open Source means that every user can deep into the source code of the programs (open source is not necesary free of charge).
  • Linux is mostly an OS kernel. It exposes computer capability (computing, memory acces and other devices) to developers. Typically C/C++ language was developed for the linux kernel
  • Distribution will configure together an OS kernel, software management (packages), an IHM (display server + descktop environement) and a bench of softwares.

Typically, ROS is designed to work on Ubuntu: Linux kernel, GNOME desktop, aptitude (apt) package manager.

Ubuntu is a good example of Open Source project:

  • It is based on another Open Source project: Debian OS to capitalize on the Debian community works, while ofering a more easy-to-use solution.
  • It searches for a profitability with Ubuntu Pro.
  • It is used in return on other Linux based OS: Pop!_os or Zorin for instance.

Visite distrowatch.com to get an idea of the galaxy of interconected Linux OSs.

The shell or Terminal

In the variety of program runing on linux, we are mostly interested in the terminal emulator, permitting to manipulate our system directly from command (moving in the directory tree, read and organize files, execute scripts or programs, manage computer resources as network, ect.).

On Ubuntu, default terminal emulator is gnome-terminal (help.gnome.org) configured with bash (on wikipedia) command interpreter.

Commands are interpretables instruction generally executing programmes. Its is exected that the following commands are kwon before to start working with ROS.
